# React Todo App - Stateful vs Stateless

## Описание
Простое Todo приложение для изучения концепции Stateful и Stateless компонентов в React. Позволяет пользователю видеть список задач и отмечать их выполнение по очереди, начиная с первой задачи в списке.

## Компоненты

### Stateful компоненты
- **App.jsx** - главный компонент приложения. Управляет состоянием списка задач (`todos`) и счётчиком выполненных задач (`completedCount`). Содержит логику удаления первой задачи при нажатии на кнопку.

### Stateless компоненты
- **TodoList.jsx** - отображает список задач. Получает массив `todos` через props и рендерит `TodoItem` для каждой задачи.
- **TodoItem.jsx** - отображает одну задачу с её ID и названием. Получает `id` и `title` через props.
- **CompleteButton.jsx** - кнопка для завершения задачи. Получает `count` (количество выполненных), `onClick` (функция-обработчик) и `disabled` (состояние блокировки). Отключается когда задач не осталось.

## Технологии
- React 18
- TypeScript
- Vite

## Установка и запуск
```bash
# Установка зависимостей
yarn install

# Запуск dev-сервера
yarn dev
```

## Что я изучил
- Разница между **Stateful** (со состоянием) и **Stateless** (без состояния) компонентами
- Использование хука **useState** для управления состоянием
- Передача данных и функций через **props**
- Типизация компонентов и props с **TypeScript**
- Рендеринг списков через `.map()` с обязательным атрибутом `key`
- Обработка событий (`onClick`)
- Деструктуризация props для чистоты кода
